# study-note-organizer

> 把原始**理工科资料**（论文 / 教材章节 / PPT / 文献 / 扫描转录文本）整理成高密度精读学习笔记，输出为 **Obsidian Markdown**。

一个用于 [Claude](https://claude.com)（Cowork / Claude Code）的 Skill。它复刻「逆向重建论文逻辑骨架」式的精读笔记风格，采用**「分部分 → 整理 → 自检 → 下一部分」**的循环工作流，逐节闭环推进，避免长资料后半段质量滑坡。

## 特性

- **循环工作流**：先通读切分，再逐部分 `整理 → 自检 → 修正 → 放行`，当前节自检通过才做下一节。
- **理工科五件套**：每个核心知识点必含 **定义 / 性质·条件 / 公式及推导（不跳步）/ 图 / 物理与数学核心理解**。
- **字数下限**：核心章 ≥600–800 字、一般小节 ≥300 字，防止总结过简（手段而非目的，不灌水）。
- **高密度风格**：层级提纲（一·/1./（1）/-）、`==高亮==` 结论、`**加粗**` 术语与数字、ASCII 示意图、批判旁注。
- **知识库补充**：可用模型自身知识补全原文缺口（前置定义、跳过的推导、成立条件等），统一用 `> [!tip] 补充` 与原文区分，绝不编造。
- **token 经济性**：reference 只读一次、自检只看当前节不回扫全文、原资料按部分读取——省重复读取/复述，留定义/推导/核心理解。

## 目录结构

```
study-note-organizer/
├── SKILL.md                    # 主体：工作流 + 风格规范 + 五件套 + 自检清单
└── references/
    └── style-guide.md          # 风格指南（含真实 before/after 样例）
```

## 安装

### Cowork（Claude 桌面端）
打开 **Settings → Capabilities**，导入本 skill 文件夹或其 `.skill` 包。

### Claude Code
把 `study-note-organizer/` 放到你的 skills 目录（项目级 `.claude/skills/` 或用户级 `~/.claude/skills/`）：

```bash
git clone https://github.com/<your-name>/study-note-organizer-skill.git
cp -r study-note-organizer-skill/study-note-organizer ~/.claude/skills/
```

## 用法

装好后，丢一份资料并说明意图即可触发，例如：

- 「把这篇 `AL31f.pdf` 按我的风格整理成 Obsidian 精读笔记」
- 「帮我消化工程热力学第 5 章，做精读笔记」
- 「整理这份文献，理工科那套展开」

输出默认存到原资料**同目录**，文件名 `<原资料名>_精读笔记.md`。

## 输出示例（片段）

```markdown
## Section 2.1　参数化方法

1. 传统参数化方法：分开建立各径向截面模型
   - 把叶面看成**大量二维区域的集合**
   - → 叶片轮廓复杂、难以平滑
   - → 控制参数过多（一排叶片上百个）   ← 缺陷
2. 表面参数化方法：显著的**低维度**特征
   - 半叶面（2003，算力约束） ——→ ==全叶面（本文）==

> [!note] 核心理解
> 物理上：把整张叶面当一个曲面+扰动面，少量控制点即可保形；
> 数学上：贝塞尔基函数加权混合，参数维度与网格密度解耦。
```

## 自定义

- **目标风格**：默认「精读学习笔记体」。如需正式综述体，可在 `SKILL.md` 调整。
- **字数下限 / 输出路径**：在 `SKILL.md` 对应小节直接改。
- **学科**：通用于理工科；`description` 里列了常见课程关键词以提升触发率，可按需增删。
