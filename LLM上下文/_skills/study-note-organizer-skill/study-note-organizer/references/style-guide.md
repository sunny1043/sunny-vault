## 1. 整体气质

Sunny 的笔记 = **论文逻辑骨架的逆向重建**。特征：
- 按原文章节顺序走（摘要 → 介绍 → Section 2 → Section 3 …），一节一个 `##`。
- 极度浓缩、省略主语、短句、信息密度高。**不是散文，是带缩进的提纲。**
- 每个方法讲完必挑毛病（缺陷 / 不足 / 适用边界）。
- 关键数字全部留下并加粗。
- 机理 / 流程 / 结构处画 ASCII 图。
- 存疑就地打括号吐槽。

## 2. 层级提纲怎么搭

四层缩进骨架（真实样例）：

```
三·六种解决 HEB 的方法及其缺陷：
    1.筛查变量——仅适用于明显不良流场的实例
    2.逐级优化——忽略了前后级的相互影响
    3.空间解耦——只适用于弱径向流场景，不适用低展弦比
    4.代理模型方法——在高维问题中很难训练，需要大量例子，成本高
    5.伴随算法——优化与变量维度无关，但容易陷入局部极值，多约束多目标场景有缺陷
    ==6.减少参数维度数——最有前景==
```

注意：`方法名——一句话点评/缺陷`，破折号后直接给判断，不展开。

## 3. 高亮与加粗的分工

- `==高亮==`：**全文最关键的结论**，一节顶多一两处。
  - 真实样例：`==减少控制变量，在保证前缘、后缘、叶体自由度的情况下有效降低维度==`
- `**加粗**`：术语首现 + **所有数字**。
  - 真实样例：`**性能**：效率 +0.48%、喘振裕度 +4.8%(10.2→15%)、压比 +0.28%、流量 +0.54%`

## 4. 批判旁注——灵魂

这是和「普通摘要」最大的区别。方法讲完不算完，要补它的边界：

```
单表面参数化：吸力面用贝塞尔面建模。
    缺陷：只建了一个面，另一面对流场的影响被忽略，且无法改变前缘角。
```

自己的疑问就地标注，别藏：

```
前一半：雇佣蜂
后一半：观察蜂（与后文相悖？）   ← 存疑直接打括号
```

在 Obsidian 里也可以升级成 callout：
```
> [!question] 这里前一半/后一半的分配和 2.2 节描述似乎矛盾，待核对
```

## 5. ASCII 示意图

「输入→输出」「机理」「结构布局」一律画图。真实样例：

数据流：
```
16 个控制点数值 (输入)
        │
        ▼
   贝塞尔面公式 = 加权混合
        │
        ▼
  叶片每点的位移 Δ (输出)
        │
        ▼
   原叶片 + Δ  →  新叶片
```

结构布局：
```
S0(IGV) | R1 S1 | R2 S2 | R3 S3 | R4 S4 S5
进口导叶   ←——— 4 个级 ———→
```

## 6. 数字抓取

数字是精读笔记的命根子。原文里每个参数、百分比、计数、耗时都要抓出来加粗。真实样例里被抓的数字：

> 种群 **100**、迭代 **8**、枯竭上限 **3**、约 **800** 次有效值；
> 效率 **+0.48%**、喘振裕度 **+4.8%**（10.2→15%）、中部最大马赫数 **1.4**；
> 共 **25 个变量**、迭代 **10** 代、约 **1000** 次（有效约 980）、约 **48 小时**。

漏数字 = 自检不通过。

## 7. 类比与口语化

降低理解门槛用大白话类比，Sunny 常这么干：
- 「压气机每一级转子叶片本质上是排"机翼"。」
- 「所有蜂在初始源附近探索，新源浓度高于原源就替换，否则继续探索。」
- 「纯靠改叶型让叶片**能在更高背压下维持流量**，从而扩了裕度。」

口语化没关系，讲清楚机理最重要。

## 8. 完整 before/after 样例

**原文（输入，论文体）：**

> The traditional parameterization method models each radial section separately, treating the blade surface as a collection of numerous 2D regions. This makes the blade profile complex and hard to smooth, and results in an excessive number of control parameters (a single blade row may have hundreds).

**整理后（Sunny 风格输出）：**

```markdown
## Section 2.1　参数化方法

1. 传统参数化方法：分开建立各径向截面模型
   - 把叶面看成**大量二维区域的集合**
   - → 叶片轮廓复杂、难以平滑
   - → 控制参数过多（一排叶片上百个）   ← 缺陷
2. 表面参数化方法：显著的**低维度**特征
   - 把叶面看成 吸力面 + 压力面 的组合
   - 半叶面（2003，算力约束） ——→ ==全叶面（本文）==
```

对照看差异：原文 3 句散文 → 笔记是带缩进的提纲；维度、年份这些**数字/事实加粗**；缺陷用 `←` 单挑出来；演进关系用 `——→`；本文方法 `==高亮==`。这就是要复刻的转写动作。
