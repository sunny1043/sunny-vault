## Section 0 — 本库性质（先读，适用于每篇笔记）

这是一个 **学习型知识库**：核心读者是**正在备考的 Sunny 本人**（不是未来的 Claude 也不是检索机器人）。因此笔记必须**人类可读、便于复习**，与通用 second-brain 的「为未来 AI 检索而写」理念不同——当本文件与 skill 默认规则冲突时，**以本文件为准**。

每篇由 Claude 写入/修改的笔记应满足：

1. **Sunny 精读五件套**（理工科笔记的硬标准）：每个知识点尽量包含
   `定义 / 性质 / 公式及推导 / 图 / 物理与数学核心理解`。这是本库既有风格，由 `assets/LLM上下文/_skills/study-note-organizer-skill` 定义，**整理资料时优先调用 `study-note-organizer` skill**。
2. **够长够细**：每章节有字数下限，宁可详尽不可过简——这是复习资料，不是摘要。
3. **公式用 LaTeX**，图优先用 **TikZJax**（精确图，已装 `obsidian-tikzjax`）或 **Excalidraw**（手绘示意图，已装）或 Mermaid（流程/框图）。图片附件统一放 `笔记/attachments/`。
4. **frontmatter 一致**：见下方「Frontmatter 规范」。
5. **双链 `[[ ]]`**：章节之间、知识点与「考研必考问」之间互链，保持知识网络可跳转。

## Section 0.5 — 动手前先核对真实状态

下结论前先读真实内容，别凭印象：
- 改/补某章前，先 `Read` 该章现有 .md，看清已有结构和风格再动笔。
- 引用考试大纲、公式、年份等外部事实时，标注来源与日期，不确定的标 `TBD`，**绝不编造公式或结论**。
- 整理资料时，先看清原始 PDF/讲义的真实内容，不要凭记忆补全。

## 命名约定

- 学科文件夹：中文学科名
- 章节文件夹：`NN-主题`（两位数字前缀，便于排序）
- 章节笔记：`第N章_主题.md` 或 `NN.M 小节名.md`
- Dashboard：放在 `assets/LLM上下文/<学科>/`，含 `<学科> MOC.md` + `Quick Reference.md`
- 题库：`学科_Practice.md`，放在 `NN-Practice/`
- 图片附件：`笔记/attachments/`，文件名可由 `image-converter` 插件自动重命名

---

## Frontmatter 规范

每篇笔记至少包含：
```yaml
---
来源: <原始资料出处，如 PDF/教材章节；无则注明>
part: <章节标识，如 ch1 / all>
整理日期: YYYY-MM-DD
keywords: <逗号分隔关键词>
tags: [<note-type>, <学科>]
---
```
note 类型：`study-note`（精读笔记）| `moc`（学科导航）| `quick-ref`（速查）| `practice`（题库）| `exam-q`（考点问答）

## 与已装工具/skill 的关系

- **整理资料 → `study-note-organizer` skill**（产出 Sunny 风格精读笔记）。
- **测验/复习 → `tutor` skill**（对本库做诊断测验、弱项训练、主动回忆）。
- **配图**：TikZJax（精确公式/坐标/电路图）、Excalidraw（手绘结构示意）、Mermaid（流程框图）、或 `baoyu-article-illustrator` / `baoyu-infographic`（AI 配图/信息图）。
- **本 second-brain skill 在本库中的角色**：导航 / 健康检查（`/obsidian-health`、`vault_health.py`）/ 跨笔记综合（`/obsidian-synthesize`）/ 可视化。**不要**用它的自动改写、自动日记、人脉/项目/看板等 PKM 功能——那些不适用于考试库。
