# Python 文件操作与异常处理

#python #file-io #exception

## 一·文件读写

一·**`open` 基本用法**：
```python
f = open("data.txt", "r", encoding="utf-8")
text = f.read()        # 一次读完
f.close()              # 必须手动关闭，否则泄漏资源
```

二·**模式**：

| 模式 | 含义 | 文件不存在 |
|------|------|:-:|
| `"r"` | 只读（默认） | 报错 |
| `"w"` | 写（覆盖） | 创建 |
| `"a"` | 追加 | 创建 |
| `"x"` | 排他创建 | 已存在则报错 |
| 加 `"b"` | 二进制（如 `"rb"` `"wb"`） | — |

三·**核心理解**：
> [!note] 核心理解
> 读写文本文件**务必指定 `encoding="utf-8"`**。不写时 Python 用系统默认编码（Windows 上常是 GBK），跨机器读中文文本几乎一定会乱码——这是 Python 中文本 IO 最密集的事故源。

---

## 二·`with` 上下文管理器（首选）

`with` 块结束时**自动调用 `__exit__` 关闭资源**，即便中途抛异常也会清理，是处理文件 / 锁 / 连接的标配。

```python
with open("data.txt", "r", encoding="utf-8") as f:
    for line in f:                # 逐行读，内存友好
        process(line)
# 离开 with 块，f 自动关闭——不用 try/finally
```

一·**逐行 vs 一次读**：
（1）`f.read()` 一次读完整个文本——小文件方便。
（2）`for line in f:` 逐行迭代——大文件内存安全首选。
（3）`f.readlines()` 一次读成**字符串列表**，每行带 `\n`。

二·**写**：
```python
with open("out.txt", "w", encoding="utf-8") as f:
    f.write("第一行\n")
    print("第二行", file=f)   # print 也能写入文件
```

三·**核心理解**：
> [!note] 核心理解
> `with` 是「资源生命周期 + 异常安全」的统一封装：对象实现了 `__enter__` / `__exit__` 两个魔法方法就能挂入 `with`，离开块时无论正常与否都执行 `__exit__`。这让「免 close、不漏写、不漏解锁」成为默认。==所有需要善后的资源，一律用 with==。

## 三·`pathlib` 路径处理

老代码多见 `os.path.join`，新代码推荐 `pathlib.Path`——面向对象的路径抽象，跨平台、链式调用。

```python
from pathlib import Path

p = Path("/Users/sunny") / "data" / "x.txt"   # 用 / 拼，跨平台
p.exists()        # 是否存在
p.is_file()       # 是不是文件
p.suffix          # ".txt"        后缀
p.stem            # "x"           无后缀名
p.parent          # 父目录 → Path
p.read_text(encoding="utf-8")        # 直接读取
p.write_text("hello", encoding="utf-8")
list(Path(".").glob("*.py"))         # 通配
```

> [!note] 核心理解
> `os.path` 是字符串拼接工具，`pathlib` 是把路径当「对象」看待：方法挂在 `Path` 上，链式表达「读 → 改 → 写」，比起 `os.path` 那一串函数式调用，可读性高出一截。

## 四·异常处理

一·**异常的本质**：异常是运行时错误的**对象**，按继承层级分类（全部继承自 `BaseException`，常用异常继承自 `Exception`）。

二·**try / except / else / finally**：
```python
try:
    value = int(input("输数字："))
except ValueError as e:        # 捕获具体类型，绑定到 e
    print(f"不是数字：{e}")
except (TypeError, KeyError):  # 可一次捕多个
    print("其他错误")
else:
    print("无异常时执行")       # ← try 成功才走
finally:
    print("无论如何都执行")      # ← 即使 except 里再 raise
```

三·**`raise` 主动抛出**：
```python
if score < 0:
    raise ValueError("分数不能为负")
```

四·**自定义异常**：继承 `Exception` 而非 `BaseException`。
```python
class InsufficientBalance(Exception):
    pass
```

---

## 五·异常处理铁律

> [!warning] 异常处理三防
> 1. **别裸 `except:`**——会吞掉 `KeyboardInterrupt` / `SystemExit` 等系统异常，掩盖 bug。要写就写 `except Exception:` 或更具体的类型。
> 2. **别捕获了不打理**——`except: pass` 让错误消失在沉默里，调试时一寸都搜不到原因。最少也要 `logging.exception()` 记一次。
> 3. **except 里不要用宽范围默默吃错**——只捕你**预期到、且能处理**的异常类型；其他的让它传播，让上层 / 日志去发现。

一·**`else` 与 `finally` 的分工**：
（1）`else` 放「成功后逻辑」——让 `try` 块只含「可能抛」的代码，异常不被本块业务逻辑混杂，结构更清。
（2）`finally` 放「必须善后」——关资源、释放锁。

二·**核心理解**：
> [!note] 核心理解
> 异常处理的设计哲学：**捕获点 = 处理点**。如果你在 `except` 里既不处理、不记录、也不 `raise`，那这个捕获就是错的——你只是在向未来的自己撒谎说「这里没事」。异常该冒泡就冒泡，直到一个真正知道怎么处理的层级。

## 六·常见异常速查

| 异常 | 触发场景 |
|------|----------|
| `ValueError` | 值类型对但取值非法（`int("abc")`） |
| `TypeError` | 类型不匹配（`"a" + 1`） |
| `IndexError` | 下标越界 |
| `KeyError` | dict 键不存在 |
| `AttributeError` | 对象没有该属性/方法 |
| `FileNotFoundError` | 文件不存在 |
| `ZeroDivisionError` | 除零 |
| `StopIteration` | 迭代器耗尽 |
| `NameError` | 名字未定义 |
| `ImportError` / `ModuleNotFoundError` | 模块导入失败 |

## 速记卡 / 一句话总结

- 文本 IO ==务必 `encoding="utf-8"`==，否则跨平台中文乱码是头号事故源。
- ==所有需要清尾的资源用 `with`==：`with open(...) as f:` 离块自动关，异常也安全。
- 读大文件用 `for line in f:` 逐行；`read()` 全读仅小文件。
- 路径用 `pathlib.Path`，新代码替代 `os.path`，跨平台 + 链式 + 直读直写。
- 异常 hierarchy 根在 `BaseException`，**自定义异常继承 `Exception`**。
- try 四段：`try` 可能抛 → `except` 捕具体 → `else` 成功才走 → `finally` 必善后。
- ==异常三防==：别裸 `except:`、别 `except: pass`、别捕不处理的异常类型。
- ==捕获点 = 处理点==：处理不了就让它冒泡，错了别向自己撒谎。
- 主动抛用 `raise ValueError("原因")`，raise 时带人类可读的说明。
