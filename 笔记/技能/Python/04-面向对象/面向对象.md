# Python 面向对象 — 类与继承

#python #oop

## 一·类与对象

一·**定义**：**类**是创建对象的「模板」，定义属性（数据）与方法（行为）；**对象（实例）**是按类实例化出来的具体个体。Python 一切皆对象，连 `int`、函数、类本身都是对象。

```python
class Dog:
    """狗。"""
    species = "Canis familiaris"     # 类属性：所有实例共享

    def __init__(self, name, age):    # 构造器：实例化时调用
        self.name = name              # 实例属性：每只狗独有
        self.age = age

    def bark(self):                   # 实例方法：第一个参数是 self
        return f"{self.name} 汪汪叫！"

d = Dog("旺财", 3)              # 实例化
print(d.name, d.bark())         # 旺财 旺财汪汪叫！
Dog.species   # 直接用类名访问类属性
```

二·`__init__` 不返回任何东西（隐式返回 `None`）；**不要写 `__init__` 返回值**。

三·`self` 就是实例自己，调用 `d.bark()` 时 Python 自动把 `d` 作为 `self` 传入。

四·**核心理解**：
> [!note] 核心理解
> 类属性 vs 实例属性：类属性是一份**全实例共享**的数据（像 `species`），改类属性会影响所有实例；实例属性每个实例**独立保留**。查找顺序是先实例属性 → 类属性，所以对**可变**类属性要格外小心——实例改它会让全实例看到效果。

## 二·继承

一·**定义**：子类继承父类，自动获得父类的属性与方法，并可**重写**或**扩展**。Python 支持**多继承**（按 MRO 解析）。

```python
class Puppy(Dog):                     # Puppy 继承 Dog
    def __init__(self, name, age, toy):
        super().__init__(name, age)   # 调父类构造器
        self.toy = toy

    def bark(self):                   # 重写
        return f"{self.name}（幼犬）汪！"

p = Puppy("豆豆", 1, "小球")
p.bark()        # 豆豆（幼犬）汪！
p.species       # 继承自 Dog
```

二·`super()` 调用父类方法，用于复用父类逻辑不重复写。

三·**多继承与方法解析顺序（MRO）**：
```python
class A:      def f(self): return "A"
class B(A):   def f(self): return "B"
class C(A):   def f(self): return "C"
class D(B, C): ...
D().f()                 # "B"——按 C3 线性化：D → B → C → A
D.__mro__               # 查看查找顺序
```

> [!warning] 易错
> 多继承虽然语法允许，但**容易引起菱形继承歧义**。新人优先用**单继承 + 组合（has-a）**，把要复用的对象作为属性持有，比多继承 dependency 更清晰、更安全。

## 三·特殊方法（魔法方法 / dunder）

以双下划线前后包裹（`__xxx__`，dunder = double underscore）的方法，由 Python 在特定时机自动调用，是自定义类与语言语义「接轨」的钩子。

| 方法 | 触发场景 | 用途 |
|------|----------|------|
| `__init__(self, ...)` | `ClassName(...)` | 构造初始化 |
| `__str__(self)` | `print(obj)` / `str(obj)` | 给人看 |
| `__repr__(self)` | `repr(obj)` / 调试 | 给开发者看（可重建） |
| `__len__(self)` | `len(obj)` | 长度 |
| `__eq__(self, o)` | `obj == o` | 判等 |
| `__lt__` `__gt__` ... | `<` `>` 比较运算 | 排序支持 |
| `__getitem__(self, k)` | `obj[k]` | 下标访问 |
| `__iter__(self)` | `for x in obj` | 迭代 |
| `__call__(self, ...)` | `obj(...)` | 实例当函数用 |
| `__enter__` / `__exit__` | `with` 语句 | 上下文管理器 |

```python
class Vector:
    def __init__(self, x, y): self.x, self.y = x, y
    def __repr__(self): return f"Vector({self.x}, {self.y})"
    def __eq__(self, o):  return self.x == o.x and self.y == o.y
    def __add__(self, o): return Vector(self.x + o.x, self.y + o.y)
    def __abs__(self):    return (self.x ** 2 + self.y ** 2) ** 0.5

Vector(1,2) + Vector(3,4)    # Vector(4, 6)
abs(Vector(3,4))             # 5.0
```

四·**核心理解**：
> [!note] 核心理解
> 「魔法方法」让自定义类**长得像内置类型**——加 `__len__` 就能 `len`，加 `__getitem__` 就能下标和切片，加 `__iter__` 就能 `for`。这是 Python **“协议而非继承”**风格的核心：不需要继承某个基类，只要实现对应方法，就被认同为符合协议（鸭子类型）。

## 四·类方法 / 静态方法 / 实例方法

```python
class Tools:
    count = 0

    def instance_method(self):            # 第 1 参数 self
        return self

    @classmethod
    def class_method(cls):                 # 第 1 参数 cls（类本身）
        Tools.count += 1
        return cls.count

    @staticmethod
    def static_method(x):                  # 无 self / cls，纯工具函数
        return x * 2
```

- **实例方法**：操作单个实例，最常用。
- **类方法 `@classmethod`**：常用于作**备选构造器**（`from_string` 之类）或操作类属性。
- **静态方法 `@staticmethod`**：和类放一起只是逻辑分组，不需 self/cls。

## 五、属性：`@property`

把「方法」伪装成「属性」读，便于在读取/校验时插逻辑而不破坏调用语法。

```python
class Student:
    def __init__(self, score):
        self._score = score           # 约定 _开头是“内部”用
    @property
    def score(self):                  # 像属性一样 s.score 读
        return self._score
    @score.setter
    def score(self, v):
        if v < 0: raise ValueError("分数不能负")
        self._score = v
```

> [!note] 核心理解
> 单下划线 `_name` 只是一种**约定**，Python 不强制私有；双下划线 `__name` 会触发**名称改写**（`_ClassName__name`），有点像「真正的私有」，但日常 **`_` 约定即可**，别用 `__`。

## 六·习题感数据建模举例

```python
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner, self._balance = owner, balance
    def deposit(self, amount):
        if amount <= 0: raise ValueError("存入须为正")
        self._balance += amount
    def withdraw(self, amount):
        if amount > self._balance: raise ValueError("余额不足")
        self._balance -= amount
        return amount
    @property
    def balance(self): return self._balance
    def __repr__(self): return f"BankAccount({self.owner}, {self._balance})"
```

调用：
```python
acc = BankAccount("Sunny", 100)
acc.deposit(50); acc.withdraw(30)
print(acc.balance, acc)        # 120 BankAccount(Sunny, 120)
```

## 速记卡 / 一句话总结

- 类属性（全实例共享）vs 实例属性（各实例独有），**查找先实例后类**。
- `__init__` 是构造器、不返回值；`self` = 实例自己，调用时自动传入。
- ==继承用 `super()` 调父类；多继承看 `__mro__`，新人优先「单继承 + 组合」==。
- dunder 是与语言语义接轨的钩子：`__repr__`/`__eq__`/`__len__`/`__getitem__`/`__iter__`/`__enter__ / __exit__`。
- ==Python 是「协议而非继承」==（鸭子类型）：实现对应方法即可被认同，不必继承基类。
- 三类方法：实例（`self`）/ 类（`@classmethod`，`cls`，常作备选构造器）/ 静态（`@staticmethod`，工具函数）。
- `@property` 把方法读成属性，可在 setter 里校验。
- 私有用单下划线约定即可，`__` 双下划线会名称改写，日常少用。
