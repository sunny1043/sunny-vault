# Python 数据结构 — 列表 / 元组 / 字典 / 集合

#python #data-structures

## 总览：四件套对比

| 类型 | 字面量 | 可变? | 有序? | 重复? | 典型用途 |
|------|--------|:---:|:---:|:---:|------|
| `list` | `[1, 2]` | ✅ | ✅ | ✅ | 顺序容器、动态数组 |
| `tuple` | `(1, 2)` | ❌ | ✅ | ✅ | 不可变记录、多返回值 |
| `dict` | `{"a": 1}` | ✅ | ✅(3.7+) | 键唯一 | 映射 / 计数 / JSON |
| `set` | `{1, 2}` | ✅ | ❌ | ❌ | 去重、集合运算 |

> [!note] 核心理解
> 「可变 vs 不可变」是这条线最重要的轴。不可变对象 = **可哈希**（能当 dict 键 / set 元素），且赋值给两个名字时**不会互相串改**。可变对象传进函数里被修改，调用方能看到——这是 Python 大多数「诡异 bug」的温床。

## 一·列表 list

`list` 是**动态数组**：有序、可变、可含异质元素，按需自动扩容。

### 创建
```python
nums = [1, 2, 3]
mixed = [1, "a", [2, 3]]      # 嵌套/异质均可
empty = []
from_range = list(range(5))    # [0, 1, 2, 3, 4]
```

### 增删改查
```python
a = [1, 2, 3]
a.append(4)        # [1,2,3,4]        尾部追加
a.insert(0, 0)     # [0,1,2,3,4]      指定位置插
a.extend([5, 6])   # [...,5,6]        合并（+= 同效）
a.pop()            # 弹出末元素；pop(0) 弹首元素（O(n)）
a.remove(2)        # 删首个等于 2 的值（不存在则报错）
del a[0]           # 按下标删
a[0] = 99          # 按下标改
a.index(99)        # 查下标；a.count(x) 查出现次数
```

### 切片（最重要）
```python
s = [0, 1, 2, 3, 4, 5]
s[1:4]     # [1,2,3]      起止左闭右开
s[:3]      # [0,1,2]
s[::2]     # [0,2,4]      步长 2
s[::-1]    # [5,4,3,2,1,0]  反转
s[-2:]     # [4,5]         取末两个
```

### 列表推导式（comprehension）
```python
squares = [x*x for x in range(5)]             # [0,1,4,9,16]
evens    = [x for x in range(10) if x % 2 == 0]
flat     = [n for row in matrix for n in row] # 二维展平
```

### 复制 = 关键坑
```python
a = [1, 2, 3]
b = a            # b 和 a 是同一个列表！
b.append(4); print(a)   # [1,2,3,4]
c = a.copy()     # 或 a[:] / list(a) —— 浅拷贝
d = copy.deepcopy(nested)   # 嵌套结构要深拷贝
```

> [!warning] 易错
> `b = a` 不是复制而是别名。要独立副本用 `.copy()` / `[:]` / `list()`，**嵌套结构**用 `copy.deepcopy`，否则改里层子列表会通过两层 alias 串改。

### 常见复杂度
| 操作 | 复杂度 |
|------|:---:|
| 索引 `a[i]`、赋值 | $O(1)$ |
| `append` / `pop()` | $O(1)$ |
| `insert(0,..)` / `pop(0)` | $O(n)$ |
| `x in a` 线性查找 | $O(n)$ |

> [!note] 核心理解
> 「尾部快、头部慢」是 list 的指纹。如果你要在头部频繁插删，改用 `collections.deque`（双端队列，两头都 $O(1)$）。

## 二·元组 tuple

`tuple` 是**不可变的 list**：定义后不能增删改。可哈希，能当 dict 键 / set 元素。

### 创建与解包
```python
point  = (3, 4)                # 单元素要加逗号 (1,)
rgb    = (255, 128, 0)
x, y   = point                # 解包
x, *_ , z = (1, 2, 3, 4)      # x=1, z=4，_ 吸收中间
((1, 2), (3, 4))              # 嵌套不变保证
```

### 为什么用 tuple 而非 list
  1. **不变性**：当常量、记录、哈希键用，安全。
  2. **多返回值**：`return x, y` 隐式返回元组。
  3. **性能**：比 list 略省内存、创建更快。

### 命名元组（升级）
```python
from collections import namedtuple
Point = namedtuple("Point", ["x", "y"])
p = Point(3, 4)
p.x            # 3，按名字访问，比 tuple[0] 可读得多
```

> [!note] 核心理解
> tuple 没有方法是因为「不可变」本身就是它的 API——它把「我不应该被改」这个意图编码进了类型。这是「类型即文档」的典范。

## 三·字典 dict

`dict` 是**哈希映射**：键值对、键唯一、3.7+ 保持插入顺序。Python 中枢数据结构。

### 创建与读写
```python
d = {"name": "Sunny", "age": 20}
d["name"]            # "Sunny"
d["city"] = "上海"    # 新增
del d["age"]          # 删除
d.get("phone", "无")  # 不存在返回默认 "无"，不报错
"name" in d           # True，查键

for k, v in d.items():   # 遍历键值
    print(k, v)
```

### 增删取改的高频陷阱
```python
d = {}
for ch in "aabbcc":
    d[ch] = d.get(ch, 0) + 1    # 计数：d["a"]=2...
# 更优写法：collections.Counter("aabbcc")
```

### 字典推导式
```python
sq = {n: n*n for n in range(5)}    # {0:0, 1:1, 2:4, ...}
```

### 合并
```python
a = {"x": 1}; b = {"y": 2}
merged = {**a, **b}     # 解包合并
merged = a | b          # 3.9+，更直观
```

### `setdefault` / `defaultdict`：省心查字典
```python
from collections import defaultdict
dd = defaultdict(list)
dd["a"].append(1)   # 不存在自动建空 list 再 append
dd["a"].append(2)   # {"a": [1, 2]}
```

> [!note] 核心理解
> dict 的核心心智是把「先检查在不在、再决定建空 / 追加」这条样板手写代码消掉：`get` / `setdefault` / `Counter` / `defaultdict` 四件武器对应四种省心场景。

## 四·集合 set

`set` 是**无序、去重的哈希容器**。`frozenset` 是其不可变版本。

### 创建
```python
s = {1, 2, 3}
s = set([1, 1, 2])     # {1, 2}    去重最常见用法
fs = frozenset({1,2})  # 不可变，可当 dict 键
```

### 运算
```python
{1,2,3} & {2,3,4}   # {2,3}  交
{1,2,3} | {3,4}     # {1,2,3,4} 并
{1,2,3} - {2}       # {1,3}  差
{1,2,3} ^ {2,3,4}   # {1,4}  对称差
2 in {1,2,3}        # True   O(1) 查询（远快于 list）
```

> [!note] 核心理解
> 判断「x 是否在一堆里」**用 set 不用 list**：list 是 $O(n)$ 线性扫，set/dict 是 $O(1)$ 哈希。规模稍大就差几个数量级。==list 做 `x in`
> 是最常见的新手性能坑==。

## 五·选择速查

| 需求 | 选什么 |
|------|------|
| 顺序、会增删顺序、按下标 | `list` |
| 不变的记录、多返回值、哈希键 | `tuple`（或 `namedtuple`） |
| 键 → 值映射 / 计数 / 缓存 | `dict`（缺省值用 `defaultdict` / `Counter`） |
| 去重 / 集合运算 / 快速存在性 | `set` / `frozenset` |
| 头部频繁插删 | `collections.deque` |
| 栈（后进先出） | `list` 的 `append` / `pop()` 即可 |
| 队列（先进先出，规模大） | `collections.deque`，不要用 list 头插 |

---

## 速记卡 / 一句话总结

- 四件套 = `list` 有序可变 / `tuple` 有序不可变 / `dict` 键值映射 / `set` 无序去重。
- ==复制 ≠ 别名==：`b=a` 是别名；浅拷贝 `.copy()` / `[:]` / `list()`；嵌套用 `copy.deepcopy`。
- ==`x in list` 是 $O(n)$，`x in set/dict` 是 $O(1)$==——存在性判断永远用 set。
- 切片 `s[a:b:c]` = 起点:止点(不含):步长，`[::-1]` 一招反转。
- 列表/字典推导式：`[f(x) for x in xs if c(x)]`。
- dict 3.7+ 保插入序；合并 `{**a, **b}` 或 `a | b`(3.9+)。
- 计数别自己写循环，用 `collections.Counter`。
- 双端操作用 `collections.deque`，list 头部插删是 $O(n)$ 坑。
- 不可变对象 = 可哈希 = 能当键。这是 tuple / frozenset 存在的根本原因。
