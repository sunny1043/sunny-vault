# 思维导图集成说明

StudyVaults 现在支持用 **Markdown 大纲直接生成交互式思维导图**，覆盖 Obsidian （写作端）和 Quartz 站点（发布端）。

---

## 1. 触发语法

在任意笔记中用 ` ```markmap ` 围栏代码块写出 Markdown 大纲：

````markdown
```markmap
# 根节点
## 二级节点
- 叶子节点
- 另一个叶子
## 另一个二级
```
````

- **Obsidian 编辑视图**：这段会显示为无语法高亮的代码块（可接受，有 `markmap` 标注）
- **Quartz 站点**：渲染为可折叠/缩放/拖拽平移的**交互式 SVG 思维导图**

---

## 2. Obsidian 端（写作期）

安装插件：**Mind Map NextGen**（james-tindal）

- 路径：`设置 → 第三方插件 → 浏览 → 搜索 Mind Map Nextgen → 安装启用`
- 状态：已安装（v1.16.0）

### 使用方式

1. 打开任意笔记（特别是 MOC / Dashboard 型大纲笔记）
2. `Cmd+P`（命令面板）→ 搜 `Mind Map`
3. 选择 **Mind Map: Open Mind Map** → 右侧出现思维导图视图

**注意**：该插件把**整篇笔记的大纲**（headings + lists）渲染为导图，而非 ` ```markmap ` 代码块。两者互补：
- 插件用于日常导航/浏览
- 围栏代码块用于站点上精确放置内联思维导图

---

## 3. Quartz 站点端（发布期）

### 原理

自定义 Quartz v5 插件 `quartz-markmap`：
- **构建时**（`npx quartz build`）：`textTransform` 在 raw Markdown 阶段用正则把 ` ```markmap ` 代码块替换为 `<div class="quartz-markmap" data-source="base64">` 占位符
- **浏览时**：页面加载后从 jsdelivr CDN 动态 import `markmap-lib` + `markmap-view`，把占位符 hydrate 成交互式 SVG

### 配置

`quartz/quartz.config.yaml`：

```yaml
- source: github:sunny1043/quartz-markmap
  enabled: true
  options:
    pan: true      # 允许拖拽平移
    zoom: true     # 允许滚轮缩放
    height: 600    # 容器高度（px）
  order: 19        # 在 syntax-highlighting(20) 之前
```

### 插件仓库

[https://github.com/sunny1043/quartz-markmap](https://github.com/sunny1043/quartz-markmap)

- 安装插件后运行 `npm run install-plugins` 即可同步

## 5. 已知限制

- `textTransform` 是正则匹配，**不解析嵌套围栏**——` ``` `` 内不能有另一个代码块
- 单页多张导图都 OK，每张独立 hydrate
- 深浅色切换：监听 `<html data-theme>` 变更后重建（SVG 文字色 + 背景跟随）
- DTS 构建偶报 `error TS2305`——不影响 ESM 输出，esbuild 的 dts 插件的并发生成时序问题
